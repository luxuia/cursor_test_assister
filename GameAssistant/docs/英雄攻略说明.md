# 英雄攻略（每英雄一条）

## 一、数据文件与结构

- **文件**：`Data/HeroGuides.json`
- **结构**：`guides` 数组，**每英雄一条**，不按对位膨胀。

### 出装（可分段或合并）

- **itemStarting**：出门装。
- **itemEarly**：前期出装。
- **itemMid**：中期出装。
- **itemLate**：后期出装。  
若以上四段都未填，则用 **itemAdvice** 作为整段出装建议。

### 技能（可分段或合并）

- **skillReleaseOrder**：技能释放顺序与连招（如「先手 E→W→R」）。
- **skillTips**：各技能释放技巧（可逐技能写要点）。  
若以上两段都未填，则用 **skillAdvice** 作为整段技能建议。

### 玩法与克制

- **playAdvice**：怎么玩好这个英雄。
- **counterAdvice**：怎么克制这个英雄（**建议写详细**：对线怎么打、团战怎么处理、出装与阵容应对、注意点等）。
- **countersList**：此英雄克制的英雄 id 列表。
- **counteredByList**：克制此英雄的英雄 id 列表。

- **heroId**：与 `Data/Dota2Heroes_FromWeb.json` 的 `id` 一致。

## 二、建议如何产出

- **己方识别到的英雄**：展示该英雄的 `itemAdvice`、`skillAdvice`、`playAdvice`。
- **敌方识别到的英雄**：展示该英雄的 `counterAdvice`，以及 `counteredByList`（可选谁克制他）。

条目数 = 英雄数量（约 128），不会因对位组合膨胀。

## 三、补全全部英雄条目

运行 **Tools/HeroGuideGenerator.MergeAllHeroes()**：

- 读取 `Data/Dota2Heroes_FromWeb.json` 得到全部英雄 id/nameCn。
- 读取现有 `Data/HeroGuides.json`（若有）。
- 对每个英雄：若已有攻略则保留，否则追加一条空内容（空字符串与空列表）。
- 写回 `Data/HeroGuides.json`。

在代码里临时调用即可：

```csharp
GameAssistant.Tools.HeroGuideGenerator.MergeAllHeroes();
```

之后在 JSON 中按英雄逐条补充 `itemAdvice`、`skillAdvice`、`playAdvice`、`counterAdvice`、`countersList`、`counteredByList` 即可。

## 四、与旧对位攻略的关系

- 原 **HeroMatchupGuides.json**（按“己方 vs 敌方”对位）已不再参与建议生成。
- 若需保留对位数据可仅作参考；当前建议只依赖 **HeroGuides.json** 的每英雄一条。
